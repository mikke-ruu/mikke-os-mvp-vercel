# MarketNote Implementation Spec 05: Finance

MarketNoteの収支ページ仕様です。D案の白ベース、細線、軽いカード、弱い影、オレンジアクセント、グリーン補助を維持しながら、ポップアップではなくページ内で収支入力できる構成にします。

## 目的

- 月単位で出店収支を確認する。
- イベントごとの売上、経費、利益を一覧で確認する。
- 必要なイベントだけ開き、売上内訳と経費内訳を入力・編集する。
- 出店詳細画面の `収支を詳しく見る →` から、該当イベントの収支カードを開いた状態で表示する。
- 本格会計ではなく、出店活動の軽い記録として扱う。

## ルート

```text
/marketnote/finance
/marketnote/finance?eventId=EVENT_ID
```

`eventId` がある場合は、対象イベントの月を表示し、そのイベントカードを開いた状態にする。

## 基本構成

1. ヘッダー
   - タイトル: `収支`
   - フィルターアイコン

2. 月切り替え
   - 前月
   - `YYYY年M月`
   - 次月

3. 月サマリー
   - 売上
   - 経費
   - 利益

4. イベントごとの収支カード
   - イベント名
   - 日付
   - 会場
   - 売上
   - 経費
   - 利益
   - `収支を編集`
   - 開閉矢印

5. 開いたカードの収支詳細
   - 売上内訳
   - 経費内訳
   - `+ 売上を追加`
   - `+ 経費を追加`
   - 利益
   - 保存

## デザイン方針

- 上部サマリーは軽いカードで3分割表示する。
- 利益はグリーン、売上と経費は黒〜グレーで表示する。
- イベントカードは軽いカードにし、閉じている時は情報を詰め込みすぎない。
- 詳細入力は開いたイベントカード内に表示する。
- ポップアップやモーダルで細かい収支入力をしない。
- 入力欄は1行中心にし、縦に重くしすぎない。

## 入力ルール

金額入力はMikke OS共通ルールに従う。

```html
type="text" inputMode="numeric" pattern="[0-9]*"
```

`type="number"` はスマホで見た目が崩れる場合があるため、基本は使わない。

## データ

MVPでは既存の `market_financial_records` を使う。

- `record_type = revenue`: 売上
- `record_type = expense`: 経費
- `title`: 内訳名
- `amount`: 金額
- `occurred_at`: 記録日。イベント日を初期値にする。
- `category`: 内訳カテゴリ。MVPではtitleと同じ値でもよい。
- `memo`: 補足。支払い情報から反映された経費では支払い方法を入れる。

出店詳細画面の支払い情報で作成された出店料レコードも、経費として同じ一覧に表示する。

## MVP範囲

- 月サマリー表示。
- イベント別収支カード表示。
- 1件だけ開いた状態で売上/経費行を編集。
- 売上/経費行の追加。
- 既存行の更新。
- 詳細画面から該当イベントカードを開く。
