# MarketNote 実装仕様 01

## ホーム / カレンダー画面仕様

対象：アオイ / Codex 実装指示用
前提：`MARKETNOTE_IMPLEMENTATION_SPEC_00.md` の共通デザインルールを必ず継承すること。

---

# 1. この画面の目的

ホーム画面は、MarketNoteの中心画面。

MarketNoteは「出店を入口にした活動ノート」として設計する。
ホーム / カレンダー画面では、以下を自然に確認できるようにする。

* これからの出店・活動予定
* 支払い状況
* 準備タスクの進捗
* 終了した活動の結果
* 振り返り
* 写真記録
* 収支の簡易確認

開催前は「予定管理・準備管理」。
終了後は「活動記録・収支・振り返り」に表示を切り替える。

---

# 2. 画面全体の構成

ホーム画面は以下の順で構成する。

1. ヘッダー
2. 月カレンダー
3. 選択日の簡易表示カード
4. 必要に応じて下部補助エリア
5. 下部ナビ

---

# 3. ヘッダー

## 表示内容

* 左上：MarketNote ロゴ / MarketNote 表記
* 右上：通知アイコン
* 月表示：例 `2025年5月`
* 月移動：左右矢印
* 今日ボタン：小さく表示

## デザイン

* D案の白ベースを継承
* 余白をきれいに取る
* ヘッダーを重くしない
* ロゴや文字は上品に見せる
* 通知アイコンは細線

---

# 4. 月カレンダー

## 基本仕様

月間カレンダーを画面上部に大きく表示する。

* 曜日行あり
* 日付セルはすべて同じ幅・同じ高さ
* 予定あり / 予定なしでセルの高さを変えない
* 文字量でセルの高さが変わらないこと
* カレンダー全体は軽い罫線で区切る
* 罫線は薄グレー
* 背景は白

## 日付セル内の基本ルール

日付セル内には、情報を詰め込みすぎない。
表示する情報は状態によって切り替える。

---

# 5. 開催前 / 当日までの通常予定表示

開催前・当日までのイベントは、予定管理向けの表示にする。

## 表示内容

* イベント名の一部
* 支払い状況
* タスク進捗

## 表示例

```text
5/22
ハンドメ...
未払い
3/5
```

または狭い場合：

```text
5/22
ハンドメ...
未
3/5
```

## 色ルール

* 出店確定：オレンジ
* 申込済み：グリーン
* 検討中：薄グレー
* 未払い：オレンジ
* 支払済：グリーン
* タスク進捗：オレンジまたは小さなテキスト
* 進捗率によって色を変えない

## 注意

開催前のセルでは、売上・経費・利益は出さない。
開催前はあくまで「準備・支払い・タスク」の確認を優先する。

---

# 6. 終了済みイベントのカレンダー表示

終了ステータスのイベントは、開催前と表示内容を切り替える。

終了後は、会場名・開催時間・支払い状況・タスク進捗を優先表示しない。
終了後は「結果」と「記録」が見える表示にする。

## 表示優先順位

1. 振り返り画像サムネイル
2. 利益金額
3. 未記録の場合は `終了 / 未記録`

## 表示パターン

### パターンA：画像あり・収支あり

```text
5/18
[小さなサムネイル]
¥20,500
```

### パターンB：画像なし・収支あり

```text
5/18
利益
¥20,500
```

### パターンC：未記録

```text
5/18
終了
未記録
```

## デザイン

* 終了済みイベントは少し控えめなトーン
* 利益金額はグリーン
* 写真サムネイルは小さく角丸
* 終了済みであることが自然に分かる
* 活動の積み上げが見えるカレンダーにする

---

# 7. カレンダー表示設定との連動

設定画面の「カレンダー表示」で、終了後の表示方法を選べるようにする。

## 選択肢

1. 写真サムネ + 利益
2. 利益のみ
3. 終了 / 未記録

## 初期値

```text
写真サムネ + 利益
```

## 補足

利益を表示したくないユーザーもいるため、設定で表示を切り替えられるようにする。
ただしMVPでは、見た目の実装を優先し、設定値の保存連動は段階的でもよい。

# 8. 選択日の簡易表示カード

カレンダーの日付をタップすると、カレンダー下に簡易表示カードを表示する。

通常予定を選択した場合と、終了済みイベントを選択した場合で表示内容を切り替える。

---

# 9. 通常予定を選択した場合の簡易表示カード

## 表示内容

* 日付
* ステータス
* 編集アイコン
* イベント名
* 開催日・時間
* 会場名
* 支払い状況
* タスク進捗
* 進捗バー

## 表示例

```text
5/22

[出店確定 ▼]  ✏

ハンドメイドフェス 2025

5/22(木) - 5/23(金) 10:00 - 17:00
東京ビッグサイト 西1・2ホール

支払い：未払い

タスク 3/5
[オレンジ進捗バー]
```

## デザイン

* D案の軽いカード
* 白背景
* 薄グレー線
* 弱い影
* 左に日付ブロックを置いてもよい
* ステータスはチップ
* 支払い状況は小さなチップ
* 進捗バーは常にオレンジ

## 操作

* ステータスは変更可能に見せる
* 編集アイコンから詳細編集へ
* カードタップで出店詳細ページへ遷移

---

# 10. 終了済みイベントを選択した場合の簡易表示カード

終了済みの日付をタップした場合、簡易表示カードは終了後モードに切り替える。

## 表示内容

* 開催日
* 小さな終了チップ
* イベント名
* 売上 / 経費 / 利益
* 収支編集アイコン
* 振り返りメモ冒頭
* 振り返り展開アイコン
* 振り返り編集アイコン
* 写真サムネイル
* 写真追加ボタン
* 写真枚数表示
* 詳細を見るボタン

## 表示例

```text
5/18　[終了]

ハンドメイドフェス 2025

売上 ¥28,000　経費 ¥7,500　利益 ¥20,500　✏

振り返り
新作の反応が良かった。次回は…　▼　✏

写真
[thumb] [thumb] [+]

2/5枚

[詳細を見る]
```

## デザイン

* 終了チップは小さく控えめ
* イベント名と収支を主役にする
* 売上・経費は黒〜グレー
* 利益はグリーンで少し強調
* 収支行の右側に鉛筆アイコン
* 振り返り行の右側に `▼` と鉛筆アイコン
* 写真サムネイルは角丸
* 写真追加は `+` ボタン
* 詳細を見るはオレンジ文字のテキストボタン、または軽いアウトラインボタン

## 写真仕様

* サムネイル表示は最大5枚
* このカード内では2〜3枚程度見せる
* `2/5枚` のように枚数表示
* 画像タップで拡大表示される想定
* `+` から写真追加できる想定

## 編集仕様

* 収支の鉛筆アイコンから収支編集へ
* 振り返りの鉛筆アイコンから振り返り編集へ
* `▼` で振り返り全文を展開
* 詳細を見るで出店詳細ページへ遷移

# 11. 下部補助エリア

カレンダー下には、必要に応じて以下を軽く表示してもよい。

* やることリスト
* 次のイベント
* 未記録の終了イベント

ただし、終了済みイベントの簡易表示カードを表示している場合は、カードを主役にする。
下部エリアは詰め込みすぎないこと。

## やることリスト

表示例：

```text
やること（期限順）

○ 出店料を支払う　5/18(日)
○ ブースレイアウトを提出　5/20(火)
○ 出展者パスを申請する　5/21(水)
```

最大3件程度。

## 次のイベント

表示例：

```text
次回イベント

5/22　ハンドメイドフェス 2025
出店確定 / 未払い / 3/5
```

最大3件程度。

* 売上ダッシュボードを大きく出す

# 14. データ連携メモ

## 通常予定表示に使うデータ

* イベント名
* 開催日
* ステータス
* 支払い状況
* チェック項目数
* チェック完了数
* 会場名
* 開始時間
* 終了時間

## 終了済み表示に使うデータ

* イベント名
* ステータス = 終了
* 売上合計
* 経費合計
* 利益
* 振り返りメモ
* 写真サムネイル
* 写真枚数

## 注意

収支は、出店詳細ページと収支ページで同じデータを使う。
ホームの終了済み簡易カードも同じ収支データを表示する。

# 16. アオイへの実装指示

まずはホーム / カレンダー画面を、D案のデザインに寄せて再構築してください。

重要なのは、開催前と終了後でカレンダー表示と簡易カードを切り替えることです。

開催前は、イベント名・支払い状況・タスク進捗を表示。
終了後は、写真サムネイル・利益金額・振り返り・写真記録を表示。

終了済みイベントは、ただの過去予定ではなく、活動の記録として見えるようにしてください。

カレンダーセルは、予定の有無や文字量に関係なく、必ず同じサイズにしてください。

下部ナビは共通仕様に合わせ、ホームアイコンをアクティブにしてください。
