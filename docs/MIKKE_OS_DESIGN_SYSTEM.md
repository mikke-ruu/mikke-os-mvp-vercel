# Mikke OS Design System

MarketNote D案を基準にした、Mikke OS共通デザインルールの初版です。
STORY / DESK / Item Studio / Order / Event などの各アプリが、別々のツールではなく同じOS内のアプリに見えることを目的にします。

## 1. デザイン思想

Mikke OSは、業務を扱うが管理画面には寄せすぎない、スマホアプリらしい軽い操作感を優先します。

- 白ベース。
- 細線。
- 軽いカード。
- 弱い影。
- オレンジアクセント。
- グリーン補助。
- ニュートラルグレー。
- 業務機能を軽く見せる。
- かわいすぎず、冷たすぎず、少し上品で実用的にする。
- 各アプリ固有の役割は残しつつ、余白、色、カード、チップ、入力欄の言語は共通化する。

## 2. カラー

色は装飾ではなく、意味を伝えるために使います。

| Token | 推奨値 | 用途 |
| --- | --- | --- |
| Primary Orange | `#ff5a1f` / `#f46a14` | 保存、追加、現在地、未払い、出店確定、主要進捗 |
| Soft Orange | `#fff6f1` / `#fff0e7` | オレンジチップ背景、選択セル背景、軽い強調 |
| Support Green | `#16833b` / `#2e7d46` | 支払済、申込済み、完了、利益、肯定状態 |
| Soft Green | `#eaf8ee` / `#f1fbf3` | グリーンチップ背景、完了タスク背景 |
| Neutral Gray | `#6f6862` / `#8a817a` / `#b8aaa0` | 検討中、不要、終了、未記録、補助テキスト |
| Background | `#fbfaf8` / `#fffdfb` | 画面背景、薄い面 |
| Border | `#e7e1dc` / `#eee9e4` | カード枠、入力枠、区切り線 |
| Text Primary | `#1f1b18` | タイトル、本文 |
| Text Secondary | `#5f5a55` / `#7b736d` | メタ情報、補助説明 |
| Danger / Error | `#c93f2d` / `#fff0e9` | 削除、エラー、注意 |

オレンジは主導線とアクティブ状態に使います。グリーンは完了、利益、肯定状態に使います。グレーは保留、対象外、終了、未記録に使います。赤系は削除やエラーに限定します。

## 3. ステータス色

MarketNoteで定義した状態を、OS全体の基準として扱います。

| 状態 | 色 | 用途 |
| --- | --- | --- |
| 検討中 | グレー | 未確定、保留 |
| 申込済み | グリーン | 前向きに進行中 |
| 出店確定 | オレンジ | 確定、現在の主状態 |
| 終了 | グレー | 完了済み、履歴 |
| 未払い | オレンジ | 対応が必要な支払い |
| 支払済 | グリーン | 支払い完了 |
| 不要 | グレー | 対象外、非アクティブ |
| 未記録 | グレー | データなし、未入力 |
| 利益 | グリーン | プラス結果 |
| エラー / 削除 | 赤系 | 注意、破壊的操作 |

押せるチップは小さな矢印、hover / active、明確なフォーカスを持たせます。表示だけのチップは矢印を出さず、薄い背景と細い枠にします。

## 4. フォント

本文とUIはシステムフォントを優先します。

- 日本語の読みやすさを優先する。
- 本文で特殊フォントを多用しない。
- ロゴやブランド表記だけ、必要に応じて別トーンを許可する。
- 数字は金額、日付、時刻が読みやすいサイズと太さにする。

推奨CSS:

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

## 5. 文字サイズ

| 用途 | 推奨サイズ | 備考 |
| --- | --- | --- |
| 画面タイトル | 20-22px | 太さは600前後 |
| セクション見出し | 15-16px | アイコンと並べる |
| カードタイトル | 16-18px | 一覧カードやサマリー |
| 本文 | 14-15px | 日本語の可読性優先 |
| 補助テキスト | 12-13px | 薄くしすぎない |
| チップ | 11-13px | 高さを抑える |
| カレンダーセル内文字 | 10-12px | 詰め込みすぎない |
| 下部ナビ | 10-12px | ラベルを出す場合のみ |

画面幅に合わせてフォントサイズを連続的に変えない。長い文字は折り返しや省略で調整します。

## 6. 角丸

D案の上品さを基準にし、丸すぎてふわふわしないようにします。

| 要素 | 推奨角丸 |
| --- | --- |
| カード | 8-12px |
| 入力欄 | 8-10px |
| チップ | 999px または 12px |
| メインボタン | 10-12px |
| サブボタン | 10-12px |
| 中央＋ボタン | 999px |
| 写真サムネイル | 8-10px |
| モーダル / メニュー | 12-16px |

大きなカードを過度に丸めない。カード内にカードを重ねすぎない。

## 7. 影

影は弱めにし、古い管理画面や重いカードに見えないようにします。

| 要素 | 方針 |
| --- | --- |
| カード影 | ごく薄い。線と併用する |
| 浮遊ボタン | 画面上で浮いて見える程度 |
| モーダル | 背景との差が分かる程度 |
| メニュー | 小さく軽い影 |

推奨例:

```css
box-shadow: 0 6px 18px rgba(31, 27, 24, 0.06);
```

濃い黒影、強いドロップシャドウ、多層の影は避けます。

## 8. アイコン

アイコンは `lucide-react` を基本にします。

| 用途 | サイズ | 線幅 |
| --- | --- | --- |
| 標準アイコン | 18-20px | 1.75-2 |
| 小アイコン | 14-16px | 1.75 |
| 下部ナビ | 22-24px | 1.75-2 |
| 中央＋ | 28-32px | 2 |

色は通常グレー、アクティブ時はオレンジ、完了や肯定状態はグリーンにします。押せないアイコンは表示しないか、明確にdisabledにします。

代表アイコン:

- ホーム: `Home`
- 一覧: `List`
- 追加: `Plus`
- 収支: `Receipt` / `ChartNoAxesColumn`
- 設定: `Settings`
- 編集: `Pencil`
- 通知: `Bell`
- カレンダー: `Calendar`
- 会場: `MapPin`
- 時間: `Clock`
- 支払い: `CircleYen`
- メモ: `NotebookText`
- 写真: `Image`
- チェック: `CheckCircle2`
- フィルター: `Funnel`

## 9. ボタン

| 種類 | 見た目 | 用途 |
| --- | --- | --- |
| Primary | オレンジ塗り、白文字 | 保存、追加、確定 |
| Secondary | 白背景、オレンジ枠、オレンジ文字 | 閉じる、戻る、補助操作 |
| Text button | 背景なし、小さめ文字 | 軽い導線 |
| Disabled | 薄いグレー、低コントラスト | 非操作 |
| Floating action button | 円形オレンジ、白い＋ | 主要追加導線 |

メインボタンは画面下部に置く場合、十分なタップ領域を確保します。押せないボタンを押せそうに見せないことを優先します。

## 10. 入力欄

入力欄は業務フォーム感を強くしすぎず、軽いカード内に自然に置きます。

| 要素 | 方針 |
| --- | --- |
| 高さ | 38-44pxを基準 |
| 枠線 | 薄いグレー |
| 背景 | 白 |
| focus | 細いオレンジ枠、薄いオレンジ背景は任意 |
| placeholder | 補助グレー |
| disabled | 薄いグレー背景、文字も控えめ |
| textarea | 高さを控えめにし、必要時だけ伸ばす |
| select | チップや軽い入力欄に寄せる |
| date / time input | 1行で見せ、アイコンを添える |

数字入力はMikke OS共通ルールとします。金額、数量、人数、枚数、時間、ブース番号など数字中心の項目はスマホで数字キーボードが出るようにします。

整数中心:

```html
<input type="text" inputMode="numeric" pattern="[0-9]*" />
```

小数が必要な可能性がある場合:

```html
<input type="text" inputMode="decimal" />
```

`type="number"` はブラウザによってスピナーや見た目崩れが出るため、基本は `type="text"` + `inputMode` を優先します。

## 11. チップ

| 種類 | 方針 |
| --- | --- |
| ステータスチップ | 状態色。押せる場合は矢印あり |
| 支払いチップ | 未払いオレンジ、支払済グリーン、不要グレー |
| 完了チップ | グリーン系 |
| フィルターチップ | 選択中だけオレンジ |
| 非操作チップ | 薄い色、矢印なし |
| 件数バッジ | 小さく、薄いグレーまたは薄いオレンジ |

押せるチップと押せないチップは必ず見分けられるようにします。押せるものはメニューや状態変更につながることを明確にします。

## 12. カード

| 種類 | 役割 |
| --- | --- |
| 基本カード | 一覧や情報表示 |
| サマリーカード | 状態確認とよく使う操作 |
| 入力カード | 新規作成やまとめ編集 |
| 終了済みカード | 履歴として控えめに表示 |
| 空状態カード | データなし + 小さな追加導線 |
| 記録カード | メモ、収支、振り返り、写真 |

カードは白背景、細い枠、弱い影を基本にします。余白はあるが縦に重くしない。情報量が多い画面では、サマリーを主役にし、編集カードは必要な時だけ開きます。

MarketNote詳細画面の役割分担:

- サマリーカード: 状態確認、ステータス変更、チェック完了切り替え。
- 編集カード: 基本情報、会場、支払い、時間などをまとめて編集。
- 記録カード: メモ、収支、振り返り、写真。

## 13. 下部ナビ

MarketNote基準:

- ホーム
- 一覧
- 中央＋
- 収支
- 設定

ルール:

- アクティブ色はオレンジ。
- 非アクティブ色はニュートラルグレー。
- 中央＋は円形オレンジで、他アイコンより大きくする。
- 影は弱め。
- ラベルは必要な場合のみ。アイコンだけでも迷わない設計を優先する。
- 新規追加、詳細編集などの入力集中画面では非表示にしてよい。

## 14. 折りたたみ

折りたたみは、情報量を軽く見せるために使います。

| 状態 | 表示 |
| --- | --- |
| 開いている | 上向き、または閉じる方向の矢印 |
| 閉じている | 下向き、または開く方向の矢印 |

初期状態の考え方:

- よく使う操作は開く。
- たまに編集する項目は閉じる。
- 記録としてよく読む項目は開く。
- 情報量が多い場合はサマリーを開き、詳細編集を閉じる。

重要ルール:

- 矢印があるなら必ず開閉できる。
- 押せないなら矢印を出さない。
- 押せる範囲は見出し行全体にする。

## 15. 空状態

空状態は、軽いカード + 短い説明 + 小さな追加導線を基本にします。

| 状態 | 表示例 |
| --- | --- |
| 予定がない日 | この日の予定はありません / ＋ この日に予定を追加 |
| データがない一覧 | まだ登録がありません / ＋ 追加 |
| 未記録 | 未記録 |
| 写真なし | 写真はまだありません / ＋ 写真を追加 |
| チェック項目なし | チェック項目はありません / ＋ 項目追加 |
| 収支なし | 収支はまだ記録されていません / 収支を詳しく見る |

空状態を大きな説明画面にしない。ユーザーが次にできる小さな行動を添えます。

## 16. 編集導線

編集できる箇所は、鉛筆アイコン、チップ、直接入力、編集モードのいずれかで示します。

ルール:

- アイコンがあるなら必ず反応する。
- 反応しないなら表示しない。
- 詳細画面では確認表示を基本にし、編集は必要な箇所だけにする。
- 保存が必要な画面では、下部のPrimaryボタンで明確に保存する。
- ステータスやチェック完了など頻繁に使う操作は、サマリー上で直接変更できるようにする。

## 17. 写真サムネイル

| 要素 | 方針 |
| --- | --- |
| サイズ | 64-80px程度 |
| 角丸 | 8-10px |
| 最大枚数 | 軽い記録エリアでは5枚を基準 |
| 追加ボタン | 点線オレンジ枠、白背景、オレンジ文字 |
| 未設定 | 失敗画像に見えない静かなプレースホルダー |

写真は画面の主役にしすぎず、記録の補助として見せます。

## 18. カレンダーセル

MarketNoteで固まったルールを基準にします。

- 日付は左上固定。
- 選択時も日付位置は動かさない。
- 選択状態は細いオレンジ枠、薄いオレンジ背景、日付丸で表現する。
- 予定あり表示は1件目だけを小さく表示する。
- 複数予定は日付横に `+1` / `+2` の小さな件数バッジを表示する。
- 支払いチップとタスク進捗はコンパクトにする。
- 終了済みはグレーや落ち着いた表示にする。
- 月表示は5週 / 6週可変にし、スマホで縦に重くしすぎない。
- 複数日イベントの期間中ライン表示は将来対応とする。

セル高さは内容で変えない。イベント名、支払いチップ、進捗、件数バッジが重ならないようにします。

## 19. 収支表示

収支は本格会計ではなく、活動記録として軽く見せます。

| 項目 | 色 |
| --- | --- |
| 売上 | 黒からグレー |
| 経費 | 黒からグレー |
| 利益 | グリーン |

詳細画面では売上、経費、利益のサマリー表示に留めます。内訳入力や詳細編集は収支ページへつなげます。スマホではポップアップで細かい収支入力をさせず、専用ページで整理します。

## 20. 今後コンポーネント化する候補

優先度1: すぐ共通化したい基礎部品。

- `Card`
- `SectionCard`
- `SummaryCard`
- `StatusChip`
- `PaymentChip`
- `PrimaryButton`
- `SecondaryButton`
- `TextButton`
- `FormInput`
- `NumericInput`
- `CollapsibleSection`
- `ProgressBar`

優先度2: 画面構成で共通化したい部品。

- `AppShell`
- `BottomNav`
- `Header`
- `EmptyState`
- `FinanceSummary`
- `TaskChecklist`

優先度3: 画面固有の調整を見ながら共通化する部品。

- `CalendarCell`
- `PhotoPicker`
- MarketNote / STORY / DESK / Item Studio / Order / Event のアプリ別サマリーカード。

共通化は急ぎすぎない。少なくとも2画面以上で同じ形が出てから抽出します。共通コンポーネントは、余白、角丸、色、focus、disabled、押せる/押せない状態の違いを吸収します。
