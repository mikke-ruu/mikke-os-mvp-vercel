# MarketNote Implementation Spec 07: Check Templates

MarketNoteのチェックテンプレート管理ページ仕様です。出店準備や当日確認でよく使うチェック項目を、自分用のテンプレートとして管理します。

## 目的

- イベントごとに毎回ゼロからチェック項目を作らなくてよいようにする。
- 追加ページ、詳細ページ、ホームの「やること（期限順）」へつながる準備項目を管理する。
- MarketNote単体の出店準備の実用性を上げる。

## ルート

```text
/settings/check-templates
```

設定トップの `チェックテンプレート` 行から遷移する。

## 画面構成

1. ヘッダー
   - 戻る
   - タイトル: `チェックテンプレート`

2. 説明カード
   - `出店準備・当日確認など、よく使うチェック項目を管理します。`

3. テンプレートカード
   - テンプレート名
   - 項目数
   - 使用中チップ
   - チェック項目一覧
   - 項目追加

4. 非表示項目
   - 非表示にした項目を戻せる。

5. 今後の接続メモ
   - 追加ページ
   - 詳細ページ
   - ホーム「やること（期限順）」

6. 保存
   - オレンジ塗りボタン。

## 初期テンプレート

テンプレート名:

```text
出店準備テンプレート
```

初期項目:

```text
出店料を支払う
ブース位置を確認
持ち物を準備
搬入時間を確認
告知文を投稿
```

## 項目データ構造

将来的にはDBで以下を持つ。

```text
title
is_default
sort_order
due_rule
is_active
```

MVPではDB変更を行わず、ブラウザ内保存から始める。

## 期限ルール

期限ルール候補:

```text
開催日当日
開催日前日
開催日の3日前
開催日の7日前
自由設定
期限なし
```

MVPでは画面上で表示・選択できるようにする。ホームの「やること（期限順）」への本格反映は後続で行う。

## MVP操作

- 項目追加。
- 項目名編集。
- 項目非表示。
- 非表示項目の復帰。
- 期限ルールの選択。
- 初期テンプレートへ戻す。
- 保存。

## 追加ページとの接続

MVPで接続する。

- 保存したテンプレートを新規出店予定追加ページで読み込む。
- アクティブな項目を初期チェック項目として表示する。
- イベント保存時に `market_check_items` として作成する。

## 詳細ページとの接続

詳細ページは、作成済みの `market_check_items` をサマリーカード内で全件表示し、その場で完了 / 未完了を切り替える。
テンプレート変更は既存イベントへ自動反映しない。既存イベントへの反映は将来対応とする。

## ホームとの接続

将来的に `due_rule` をもとに各チェック項目へ `due_date` を設定し、ホームの「やること（期限順）」へ表示する。

MVPでは、期限がない場合はイベント開催日を仮期限として扱う。

## 後続で対応すること

- 複数テンプレート。
- 並び替え。
- 期限ルールのDB保存。
- テンプレートごとの使い分け。
- イベント種別ごとのテンプレート切り替え。
- 設定内容のSupabase保存。
- 既存イベントへテンプレートを適用する操作。
