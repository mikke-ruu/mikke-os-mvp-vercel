# MarketNote Implementation Spec 06: Settings and App Switcher

MarketNoteの設定画面と、三本線から開くMikke OSアプリ切り替えモーダルの仕様です。
`MIKKE_OS_DESIGN_SYSTEM.md` を参照し、D案の白ベース、細線、軽いカード、弱い影、オレンジアクセント、グリーン補助を維持します。

## 目的

- MarketNoteを自分の運用に合わせて調整できる場所を作る。
- チェックテンプレート、収支カテゴリ、支払い方法、カレンダー表示、通知設定を集約する。
- MarketNoteからSTORY / DESKへ自然につながる導線を作る。
- スマホのホーム画面追加導線を置く。
- 三本線からMikke OS内のアプリを切り替えられるようにする。

## ルート

```text
/settings
```

MVPでは既存の設定ルートを使う。将来的にMarketNote固有設定を分ける場合は `/marketnote/settings` も候補にする。

## 設定画面の構成

1. ヘッダー
   - 左: 三本線メニュー
   - 中央: `設定`
   - 右: 通知アイコン

2. プロフィールカード
   - 表示名
   - ハンドル
   - 活動カテゴリ / 説明
   - 活動エリア
   - STORYと共通プロフィールである説明
   - `STORYで編集する`
   - `STORYを開く`

3. MarketNoteの設定
   - チェックテンプレート
   - 収支カテゴリ
   - 支払い方法
   - カレンダー表示
   - 通知 / リマインダー

4. Mikke OS連携
   - STORY
   - DESK

5. アプリをホーム画面に追加
   - MarketNote
   - STORY
   - DESK

6. その他
   - データ管理
   - ヘルプ
   - ログアウト

## MVPで実装する範囲

- 設定トップの見た目。
- プロフィールカードの表示。
- MarketNote設定項目の一覧表示。
- カレンダー表示のみ、折りたたみで選択UIを表示。
- STORY / DESK連携カード。
- ホーム画面追加の見た目。
- ログアウト。
- 三本線からアプリ切り替えモーダルを開く。

## 後続で本格実装する範囲

- チェックテンプレートの作成、編集、並び替え、非表示。
- 収支カテゴリの作成、編集、並び替え、非表示。
- 自由入力カテゴリを次回候補にする設定。
- 支払い方法の追加、編集、並び替え、非表示。
- 通知 / リマインダーの実処理。
- PWAホーム画面追加の実処理。
- STORY / DESKの実連携設定。

## チェックテンプレート

将来的には以下を管理する。

- テンプレート名。
- チェック項目。
- 並び順。
- 期限ルール。
- 非表示。

期限ルール例:

- 開催日当日。
- 開催日前日。
- 開催日の3日前。
- 自由設定。

## カレンダー表示

設定トップ内で折りたたみ表示する。

終了後の表示:

- 写真サムネ + 利益
- 利益のみ
- 終了 / 未記録

初期値は `写真サムネ + 利益` とする。

## 三本線アプリ切り替えモーダル

共通コンポーネント:

```text
components/MikkeAppSwitcher.tsx
```

役割:

- Mikke OS内のアプリを切り替える。
- 現在開いているアプリを分かるようにする。
- 連携済み / 未連携を表示する。
- 将来のItem Studio / Order / Event / Studio / Connectに拡張できるようにする。

MVP表示:

- MarketNote
- STORY
- DESK

表示場所:

- 設定画面ヘッダー。
- MarketNoteホーム。
- MarketNote一覧。
- MarketNote収支。
- 追加ページ、詳細ページなどの入力集中画面では、戻る/保存導線を優先するため表示しなくてよい。

デザイン:

- スマホでは下から開くボトムシート。
- 白背景。
- 角丸カード。
- 弱い影。
- 現在アプリは薄いオレンジで強調。
- 連携済みはグリーンチップ。
- 外側タップまたは閉じるボタンで閉じる。

## 重要ルール

- 押せないアイコンや矢印は出さない。
- 折りたたみ矢印がある場合は必ず開閉できる。
- 未実装の詳細管理は、押せるように見せすぎない。
- 設定画面は項目を詰め込みすぎず、セクションごとに軽いカードで整理する。
- プロフィール編集はMarketNote内で抱え込まず、STORYの共通プロフィール管理へつなげる。
