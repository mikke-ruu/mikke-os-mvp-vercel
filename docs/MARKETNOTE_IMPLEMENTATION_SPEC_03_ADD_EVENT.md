# MarketNote Implementation Spec 03

## 出店予定追加ページ仕様

対象: アオイ / Codex 実装用

この画面は、MarketNoteを「見るだけ」から「予定を登録できる」アプリにするための入力画面。中央＋ボタンとホームのカレンダー日付から開ける。

## 目的

- 新しい出店予定をスマホで軽く登録できるようにする。
- 必須入力はイベント名と日付だけにする。
- D案の世界観を維持する。
- 詳細ページの編集UIへ展開しやすい構造にする。

## デザイン方針

- 白ベース。
- 細線。
- 軽いカード。
- 弱い影。
- オレンジアクセント。
- グリーン補助。
- ニュートラルグレー。
- 入力フォームも業務管理画面っぽくしない。
- スマホでさっと登録できる軽さを優先する。

参考方向:
- 添付画像のように、白いカードを縦に並べる。
- アイコンは細線で、セクション名の横に小さく置く。
- 支払い追加、項目追加、閉じるは白背景 + オレンジ枠/文字。
- 保存ボタンはオレンジ塗り。

## ルートと導線

### 中央＋ボタンから追加

- 遷移先: `/marketnote/new`
- 開催日は未入力。
- ユーザーがイベント名と開催日を入力する。

### カレンダーの日付から追加

ホームのカレンダーで日付を選択した時、その日に予定がない場合はカレンダー下に軽い空状態カードを表示する。

表示例:

```text
5/24（土）

この日の予定はありません

＋ この日に予定を追加
```

ボタン押下時:

```text
/marketnote/new?startDate=2025-05-24
```

追加ページ側では `startDate` がある場合、開始日/開催日の初期値として入れる。`startDate` がない場合は通常の未入力状態で開く。

予定がある日にも、将来的には以下の導線を表示できるようにする。

```text
＋ この日に別の予定を追加
```

MVPでは、まず予定がない日の「この日に予定を追加」だけでよい。

## 必須項目

- イベント名
- 開催日、または開始日

それ以外は未入力保存可。

## 画面構成

1. ヘッダー
   - タイトル: `出店予定を追加`
   - 戻る導線
   - 閉じる導線

2. 基本情報
   - イベント名: 必須
   - 日付: 必須
   - 複数日イベント切り替え
   - ステータス: `検討中` / `申込済み` / `出店確定`

3. 日時詳細
   - 折りたたみ可
   - 開始時間
   - 終了時間
   - 集合時間
   - 撤収時間

4. 会場情報
   - 折りたたみ可
   - 会場名
   - 住所

5. 支払い情報
   - 折りたたまない
   - 支払い状況: `未払い` / `支払済` / `不要`
   - 支払い方法: `現金` / `QR` / `カード` / `ポイント` / `その他`
   - 金額
   - `+ 支払い追加`

6. メモ
   - 折りたたまない
   - URL専用欄は作らず自由記述
   - 詳細ページでも編集できる前提

7. チェック項目
   - 折りたたみ可
   - 共通テンプレート選択
   - 項目追加
   - MVPでは見た目優先でOK

8. 保存ボタン
   - オレンジ塗り
   - ラベル: `出店予定を保存`

9. 閉じるボタン
   - 白背景
   - オレンジ枠
   - オレンジ文字

## 複数日イベント対応

初期表示はシンプルに見せる。

```text
開催日  2025/5/22
□ 複数日イベント
```

複数日イベントをONにした場合:

```text
開始日  2025/5/22
終了日  2025/5/23
```

内部的には、1日イベントでも複数日イベントでも以下の形を想定する。

```text
start_date
end_date
```

1日イベントの場合:

```text
start_date = 2025-05-22
end_date = 2025-05-22
```

複数日イベントの場合:

```text
start_date = 2025-05-22
end_date = 2025-05-23
```

こうしておくと、あとでカレンダー表示や一覧表示を拡張しやすい。

## MVPでの保存方針

現行テーブルでは `market_events.event_date` が開始日の役割を持つ。

- `start_date` 相当: `market_events.event_date`
- `end_date` 相当: MVPでは `market_events.private_note` に保存
- 複数日フラグ: MVPでは `market_events.private_note` に保存
- 会場名: `market_events.venue_name`
- 住所: `market_events.area`
- メモ: `market_events.public_note`
- 支払い金額: `market_financial_records`
- 支払い状態: `market_financial_records.payment_status`
- 支払いチェック: `market_check_items`

将来のDB拡張時は `market_events` に `start_date` / `end_date` カラムを追加し、詳細ページ、カレンダー、一覧で同じ日付構造を使う。

## カレンダー表示の将来方針

複数日イベントの場合、将来的には以下の表示にする。

- 開始日セルにイベント名、支払い、タスク進捗を表示。
- 期間中の日付には薄いオレンジ背景や細いラインで「続いている予定」と分かる表示。
- 終了日までイベント期間として認識。

MVPでは、まず開始日にだけ表示でよい。ただし、データ設計は最初から `start_date` / `end_date` を想定する。

## 実装メモ

- 既存の `createMarketEvent`, `addCheckItem`, `addFinancialRecord` を使う。
- スキーマ変更はこの段階では行わない。
- 追加ページ内のフォーム部品は、詳細ページ編集へ展開しやすい単位に分ける。
- 中央＋からも、カレンダー日付からも同じ `/marketnote/new` を使う。
- `npm.cmd run lint` と `npm.cmd run build` で検証する。
